# VELOTIME — Project Context
React 19 + Vite 8 application rebuilt from static HTML pages generated by Emergent.
The goal is to recreate a clean, modular, scalable frontend for a Strava-segment race results platform.

---

## 🏗 Tech Stack
- React 19 (use API, Actions, modern JSX)
- Vite 8 (fast dev server + build)
- JavaScript (no TypeScript unless explicitly requested)
- React Router (client-side routing)
- CSS modules or global CSS (no Tailwind unless requested)
- No UI libraries by default (components handcrafted)

---

## 🎯 App Purpose
VeloTime is a web app for:
- browsing Strava-segment race events
- viewing event details
- showing leaderboards and results
- authenticating users (login/register)
- optionally integrating with Strava API

The UI style is clean, minimal, modern, similar to Emergent-generated pages.

---

## 📁 Project Structure (feature-based)

src/
  api/                → API calls (Strava integration, mock services)
  assets/             → images, global styles
  components/
    common/           → reusable UI (Navbar, Footer, Button, Input)
    features/
      events/         → EventCard, EventList, EventDetail
      results/        → ResultsTable, Leaderboard, SegmentInfo
  context/            → AuthContext, global state
  hooks/              → useEvents, useResults, useAuth
  pages/              → Home, Events, EventDetails, Login, Register
  services/           → business logic, data transforms
  utils/              → helpers (formatting, date utils)
  App.jsx             → router + layout
  main.jsx            → React entry point

## 🧠 Gemini CLI Usage Guidelines
When generating files:
- Always generate **one file at a time**
- Follow the folder structure above
- Use React 19 conventions
- Keep components small and modular
- Avoid unnecessary dependencies
- Use functional components only
- Use modern JSX (no `import React from 'react'`)

---

## 📝 Naming Conventions
- Components: PascalCase (EventCard.jsx)
- Hooks: camelCase starting with “use” (useEvents.js)
- Pages: PascalCase (Events.jsx)
- Context: PascalCase (AuthContext.jsx)
- Utils: camelCase (formatDate.js)
